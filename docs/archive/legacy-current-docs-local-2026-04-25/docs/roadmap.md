# Stormhelm Roadmap

## Phase 1

- separate background core and PySide6 UI
- local HTTP IPC
- SQLite persistence
- bounded concurrent job manager
- safe starter tools
- chat MVP
- tray-ready shell
- packaging-ready repo structure

## Phase 2

- real model adapter layer
- richer planning/orchestration pipeline
- better session management
- persistent event streaming instead of polling-only UI updates
- startup service strategy for Windows
- settings editing from the UI
- improved note and memory retrieval workflows

## Phase 3

- voice input pipeline
- speech output adapter
- wake-word-ready architecture
- background idle listening controls

## Phase 4

- browser automation
- app launching and foreground control
- guarded desktop automation
- stronger permissions and approval workflows

## Phase 5

- semantic memory retrieval
- embedding storage or vector extension
- screen/computer-use integration
- multimodal context handling
