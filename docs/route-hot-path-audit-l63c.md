# Stormhelm L6.3C Route Hot-Path Audit

Phase L6.3C applies one route latency doctrine across Stormhelm: every route family must declare whether it is `instant`, `cached_fast`, `bounded_live`, `async_continuation`, `clarification`, or `unsupported`. Ghost/default routes may not do unbounded live work. Deck/detail routes may explicitly request fuller probes, but they still need named budgets, trace fields, and truthful deferred states.

## L6.3C Conversion Summary

- `src/stormhelm/core/latency.py` now declares route-family latency contracts and exposes `RouteLatencyPosture`, `RouteFamilyLatencyContract`, and `get_route_latency_contract`.
- System/resource routes now emit shared cache/defer fields: `system_cache_hit`, `system_cache_age_ms`, `system_freshness_state`, `live_probe_deferred`, `live_probe_job_id`, and `live_probe_timeout_ms`.
- CPU/RAM/resources now use cached/fast baseline behavior on Ghost/default and defer hardware helper refresh.
- Battery/power now uses fast baseline/cache on Ghost/default; charge-rate and hardware detail stay in live/detail paths.
- Storage and network status now use cached/deferred defaults and background refresh.
- Weather and location now share the doctrine: cached/fresh data returns quickly, missing/stale data returns truthful deferred state, and slow live work is moved to background refresh unless Deck/detail explicitly asks.
- Recent files now returns cached/deferred compact results instead of scanning allowlisted roots on the Ghost hot path.
- `scripts/live_runtime_latency_trace.py` now reports expected/actual posture, budget exceedance, blocking live-probe detection, cache hit/miss, async deferred state, payload size, and status/snapshot impact.

## Remaining Gaps

- Network diagnosis, app/window/system control, software verification/recovery, Discord relay, and deep file search now have explicit policy posture but still need deeper per-action budget instrumentation.
- Hardware telemetry has cache/defer protection through resources/power, but a dedicated worker progress model would make full Deck telemetry clearer.
- Workspace restore/search continuation exists at the route-policy level; per-tool scan spans should be expanded for future trace precision.
